# MVTec Screw - Binary Classification with ResNet18

## Overview

This repository contains an end-to-end notebook pipeline for binary classification on the MVTec screw dataset:

- Class 0: good
- Class 1: not-good

The workflow is implemented in [main_analysis.ipynb](main_analysis.ipynb) with a clean, recruiter-friendly structure:

- Centralized imports in the first code cell
- Concise outputs focused on metrics and visualizations
- Brief professional comments per cell
 
The notebook is now designed to be reliable for end users:

- Training is forced on every run so fold metrics, ROC curves, and confusion matrices are always produced.
- If dataset folders are missing, data is downloaded automatically.
- The best fold is saved to `best_model.pth` at the end of each run, so the artifact is generated by training rather than downloaded.
- Large dataset/model transfers are handled with `gdown` (resume mode enabled in the download script).

## Current Pipeline

- Backbone: ResNet18 (ImageNet pretrained when available)
- Input resolution: 256 x 256
- Data augmentation (train): rotation +/-15 degrees, horizontal/vertical flips
- Validation transform: resize + normalization only
- Loss: CrossEntropyLoss with class weights
- Optimizer: Adam
- Evaluation: 5-fold stratified cross-validation
- Early stopping: enabled with configurable patience

## Setup

1. Create and activate a virtual environment.

```bash
python -m venv .venv
source .venv/bin/activate
```

2. Install dependencies.

```bash
pip install -r requirements.txt
```

The notebook also checks for missing Python packages on first run and installs the minimum dependencies automatically if needed.

## Dataset Download

Download and extract the dataset with:

```bash
python download_data.py --output-dir ./data
```

Useful options:

```bash
# Dataset only
python download_data.py --dataset-only --output-dir ./data
```

Expected training folders used by the notebook:

- data/mvtec_screw/train/good
- data/mvtec_screw/train/not-good
- data/mvtec_screw/test

## Run the Notebook

```bash
jupyter lab main_analysis.ipynb
```

At runtime, the notebook follows this sequence:

1. Validate dataset folders under `data/mvtec_screw`.
2. If missing, run `download_data.py --dataset-only` automatically.
3. Run full 5-fold stratified cross-validation training.
4. Save the best fold to `best_model.pth`.
5. Run summary plots and test-image inference.

Main outputs produced in the notebook:

- EDA charts (sample grid, class distribution, intensity analysis)
- Per-fold confusion matrices
- Per-fold ROC curves
- Cross-validation summary metrics
- Qualitative test-grid predictions

## Saved Model Artifact

The notebook selects the best fold by validation accuracy and saves weights to:

- best_model.pth

`best_model.pth` is regenerated automatically at the end of every training run.

## Notes

- The notebook auto-detects CPU/GPU.
- Dataset paths are validated early and auto-downloaded when needed.
- Test images are treated as unlabeled for qualitative inference.
- Model loading uses `map_location='cpu'` for safer portability across environments.

## Troubleshooting

- If download fails, manually download from Google Drive and extract under data/mvtec_screw.
- If memory is limited, reduce batch size and/or number of folds/epochs in notebook config.
